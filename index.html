Memory en Java

Jeu du Memory en Java (avec Scene Builder)

Description :

Le projet est un jeu du Memory où l'objectif est de retrouver les paires de cartes identiques. Chaque carte a une image cachée, et les cartes sont mélangées à chaque début de partie. Lorsque l'utilisateur clique sur une carte, elle se retourne. Si deux cartes retournées sont identiques, elles restent visibles. Le jeu se termine lorsque toutes les paires sont trouvées.

Fonctionnement :

  • Début du jeu : Une grille de cartes est affichée, toutes retournées (cachées), créée avec Scene Builder.
  • Interaction : L'utilisateur clique sur une carte pour la retourner et voir son image.
  • Comparer les cartes : Lorsqu'une deuxième carte est retournée, les deux cartes sont comparées. Si elles sont identiques, elles restent visibles, sinon elles se retournent après un court délai.
  • Victoire : Le jeu est gagné lorsque toutes les paires sont trouvées. Un message de victoire est affiché.
  • Réinitialisation : L'utilisateur peut recommencer le jeu en cliquant sur un bouton de réinitialisation.

Veliko

Projet Veliko

Objectif

Le projet permet d'interagir avec l'API Vélib' pour récupérer des informations sur les stations de vélos en libre-service, puis de stocker ces données dans une base de données MySQL pour les réutiliser et les afficher.

Fonctionnement

  • Récupération des données : L'API Vélib' fournit des informations en temps réel sur les stations de vélos : nombre de vélos disponibles, état des bornes, etc.
  • Stockage dans MySQL : Les données récupérées sont stockées dans une base de données MySQL via Symfony et Doctrine, incluant des informations sur l'emplacement des stations et leur statut.
  • Réutilisation des données : Les données stockées peuvent être consultées pour générer des rapports ou être affichées en temps réel sur un site ou une application.

Technologies utilisées

  • PHP & Symfony : Pour la gestion de la logique, l'interaction avec l'API, et le stockage des données.
  • MySQL : Pour stocker et organiser les informations relatives aux stations Vélib'.

Source de l'API

Les données proviennent de l'API officielle Vélib', accessible ici : https://www.velib-metropole.fr/donnees-open-data-gbfs-du-service-velib-metropole

Memory en JS

Explication du jeu du Memory en JS

Description :

Le projet est un jeu du Memory interactif où l'objectif est de retrouver les paires de symboles identiques parmi un ensemble de cartes. Chaque carte présente un symbole caché (émoticônes), et les cartes sont mélangées à chaque début de partie. Lorsqu'un joueur clique sur une carte, elle se retourne. Si deux cartes retournées sont identiques, elles sont considérées comme une paire et restent visibles. Le jeu se termine lorsqu'une paire est trouvée pour chaque symbole.

Fonctionnement :

  • Initialisation : Un tableau d’émoticônes est défini et dupliqué pour créer les paires.
  • Mélange : Les symboles sont mélangés aléatoirement avant de les afficher dans l'interface.
  • Interaction : L'utilisateur clique sur les cartes pour les retourner. Si les symboles correspondent, les cartes restent visibles, sinon elles se retournent après un court délai.
  • Fin du jeu : Le jeu se termine lorsqu'il n'y a plus de cartes à retourner, et un message indique la victoire.

Technologies :

  • HTML/CSS : pour la structure et le design visuel.
  • JavaScript : pour la logique du jeu (mélange des cartes, gestion des interactions, détection des paires).

Casse brique en JS

Jeu Casse-Brique

Détruisez des briques avec une balle et une raquette. Un jeu simple et fun !

1. Structure du Jeu

  • Canevas : Zone de jeu de 480x320px avec fond sombre.
  • Composants :
    • Balle : Se déplace et rebondit.
    • Raquette : Contrôlée avec les touches fléchées.
    • Briques : Disparaissent quand elles sont touchées par la balle.

2. Mécaniques du Jeu

  • Balle : Rebonds sur les bords et la raquette.
  • Raquette : Déplacement à gauche et à droite avec les touches fléchées.
  • Conditions de victoire et de perte :
    • Victoire : Toutes les briques sont détruites.
    • Perte : La balle tombe sans être rattrapée par la raquette.

3. Détection des Collisions

  • Collisions Balle-Briques : La balle touche et détruit les briques.
  • Collisions Balle-Raquette : La balle rebondit si elle touche la raquette.
  • Fin du Jeu : Si la balle touche le bas sans la raquette, le jeu se termine.

4. Contrôles

  • Flèche droite : Déplacer la raquette vers la droite.
  • Flèche gauche : Déplacer la raquette vers la gauche.

5. Interface et Notifications

  • Message de fin : Un message "Perdu !" apparaît lorsque la balle tombe en bas.
  • Redémarrage : Cliquer sur "Perdu !" pour recommencer le jeu.

6. Code en Bref

  • Canvas : Zone graphique pour afficher le jeu.
  • Boucle de jeu : Mise à jour continue de la balle, de la raquette et des briques.
  • Interaction avec le clavier : Détection des touches pour déplacer la raquette.
  • Collisions : Détection des collisions entre la balle, la raquette et les briques.

Présentation du code : Planning interactif avec validation de formulaire

Ce code permet à un utilisateur de remplir un planning de la semaine en saisissant une matière, un jour et une heure via un formulaire. Il comprend une validation dynamique des champs, avec des messages d'erreur si un champ est vide et des messages de succès lorsque la saisie est correcte.

Cette approche assure une expérience utilisateur fluide, tout en validant les informations avant de les soumettre.

Bourse

Scraper d'Informations sur les Actions Françaises

L'application permet de scraper les données des actions françaises depuis des pages web et de les utiliser et comparer facilement. Grâce à Tkinter, une interface graphique interactive est proposée pour visualiser et manipuler les informations boursières.

Fonctionnalités principales :

  • Extraction automatique : Je récupère les informations financières des actions françaises sur des sites de marché.
  • Affichage dynamique : Les données sont affichées dans un tableau, avec possibilité de filtrer et sélectionner des éléments.
  • Comparaison et gestion : Je peux ajouter des actions à une liste d'éléments choisis pour les comparer ou les retirer du tableau principal.

Comment ça fonctionne ?

  1. L'application scrappe les données d’une page web dédiée aux actions françaises.
  2. Les informations sont présentées dans un tableau interactif où je peux facilement filtrer, sélectionner et organiser les actions.
  3. Je peux comparer des actions en les sélectionnant et en les ajoutant à une liste séparée pour les analyser.

Cette application me permet de récupérer et d'analyser rapidement des données boursières sur les actions françaises.
